# 变更日志

所有重要的项目变更都将记录在此文件中。

## [1.3.0] - 2026-03-18

### ✨ 新增功能
- **用户配置持久化**：配置保存到用户数据目录，避免信息泄露
  - 配置文件位置：`%APPDATA%/calendar-feishu/feishu-user-config.json`
  - 首次启动自动弹出配置向导，引导用户配置飞书凭证
  - 配置信息不打包到 release 中，避免反编译泄露
  - 开发时可使用 `feishuConfig.example.ts` 作为参考
- **配置管理功能**：
  - 飞书日历管理中心添加 "⚙️ 飞书配置" 按钮
  - 支持随时修改 App ID、App Secret 和 Calendar ID
  - 配置保存后自动刷新，无需重启应用
  - Calendar ID 为可选项，可稍后在飞书日历管理平台创建
- **隐藏终端启动**：提供 VBScript 脚本，双击即可启动开发服务器，终端不显示在任务栏
  - `start-dev.vbs` - 隐藏终端启动（无日志）
  - `start-dev-logging.vbs` - 隐藏终端启动（带日志记录）
  - `start-dev.ps1` - PowerShell 隐藏启动
  - `创建桌面快捷方式.bat` - 一键创建桌面快捷方式
- **VS Code 任务配置**：集成后台任务启动，支持快捷键启动
- **飞书全量同步按钮**：在飞书日历管理中心添加"📥 从飞书同步"按钮
  - 支持从飞书全量下载日历日程
  - 智能更新已存在的日程
  - 自动清除对应日历的同步队列

### 🔄 优化改进
- **修复同步问题**：
  - 修复同步日程时使用默认配置而非用户配置的问题
  - 修复日历 ID 未动态获取的问题
  - 新增日程、更新日程、删除日程均可正确同步到飞书
  - 所有 IPC 处理器（createEvent、updateEvent、deleteEvent）均使用用户配置
- **简化同步逻辑**：重构 SyncManager，避免重复同步问题
  - 启动时只添加本地独有事件（无 feishuEventId）到队列
  - 移除 incrementalSync 方法，不再重复检测本地独有事件
  - sync() 方法简化，只返回队列统计
- **同步队列优化**：优化队列管理，只在本地操作时添加到队列
  - 新增日程 → 自动加入 create 队列
  - 编辑日程 → 自动加入 update 队列
  - 删除日程 → 自动加入 delete 队列
- **配置检测优化**：
  - 首次启动不再强制弹窗，改为顶部提示条
  - 提示条可关闭，不影响正常使用
  - 可在飞书日历管理中心随时打开配置向导
- **文档整合**：整合启动说明文档，提供完整的使用指南
