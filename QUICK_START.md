# 🚀 快速启动指南

## 💡 最推荐的方式

### 方式 1：双击启动（隐藏终端）⭐

**操作步骤**：
1. 双击项目根目录的 `start-dev.vbs` 文件
2. 开发服务器会在后台启动
3. 打开浏览器访问 `http://localhost:5173`

**优点**：
- ✅ 终端不显示在任务栏，不会被误关闭
- ✅ 双击即可，简单方便
- ✅ 完全隐藏，不占用桌面空间

**停止方法**：
- 按 `Ctrl+Shift+Esc` 打开任务管理器
- 找到 `node.exe` 进程
- 右键 → 结束任务

---

### 方式 2：VS Code 任务启动（开发推荐）⭐⭐⭐

**操作步骤**：
1. 在 VS Code 中按 `Ctrl+Shift+P`
2. 输入 `Tasks: Run Task`
3. 选择 `npm: dev`

**优点**：
- ✅ 可以查看完整的控制台输出
- ✅ 集成在 VS Code 中，不会被误关闭
- ✅ 支持错误检测和日志查看

**停止方法**：
- 在 VS Code 终端中按 `Ctrl+C`

---

### 方式 3：传统终端启动

**操作步骤**：
1. 在项目根目录打开终端
2. 运行 `npm run dev`

**优点**：
- ✅ 可以看到完整的日志输出
- ✅ 方便调试

**缺点**：
- ❌ 终端窗口显示在任务栏
- ❌ 容易被误关闭

---

**方法 1**：使用带日志的版本
```bash
双击 start-dev-logging.vbs
# 日志保存在 dev.log
```

**方法 2**：重定向输出
```bash
npm run dev > dev.log 2>&1
```

**方法 3**：使用 VS Code
- 打开 VS Code
- 按 `Ctrl+Shift+P`
- 选择 `Tasks: Run Task` → `npm: dev`

**步骤**：
1. 按 `Win+R`
2. 输入 `shell:startup`
3. 将 `start-dev.vbs` 的快捷方式复制到打开的文件夹

**注意**：开发阶段不推荐开机自启

## ❓ 常见问题

### Q1: 双击后没有任何反应？
**A**: 这是正常的，服务已经在后台启动。可以：
1. 打开浏览器访问 `http://localhost:5173`
2. 或查看任务管理器确认 `node.exe` 进程

### Q2: 如何确认服务是否启动成功？
**A**: 
1. 浏览器访问 `http://localhost:5173`
2. 命令行运行：`netstat -ano | findstr :5173`

### Q3: 端口被占用怎么办？
**A**: 
1. 找到占用端口的进程：`netstat -ano | findstr :5173`
2. 结束进程：`taskkill /F /PID <进程 ID>`
3. 或修改端口：编辑 `vite.config.ts`

### Q4: 脚本无法运行？
**A**: 
1. 确保已安装 Node.js 和 npm
2. 确保在项目根目录运行
3. 检查 PowerShell 执行策略（仅 PowerShell 脚本需要）

### Q5: 如何修改启动端口？
**A**: 
编辑 `vite.config.ts` 或 `.env` 文件：
```typescript
server: {
  port: 5173  // 修改这里的端口号
}
```

## 🔍 进程管理

### 查看进程
```bash
# 查看所有 node 进程
tasklist | findstr node

# 查看特定端口的进程
netstat -ano | findstr :5173
```

### 结束进程
```bash
# 结束所有 node 进程
taskkill /F /IM node.exe

# 结束特定 PID 的进程
taskkill /F /PID <进程 ID>
```

### 图形化方式
1. 按 `Ctrl+Shift+Esc` 打开任务管理器
2. 找到 `node.exe` 进程
3. 右键 → 结束任务
