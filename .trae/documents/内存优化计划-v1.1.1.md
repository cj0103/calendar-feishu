# 内存优化计划 (v1.1.1)

## 当前状态
- **内存占用**: 120-170MB
- **目标**: 优化至 80-100MB 以下
- **待解决问题**: 终端 Autofill 错误提示

## 问题分析

### 1. Autofill 错误提示
**错误信息**:
```
Request Autofill.enable failed. {"code":-32601,"message":"'Autofill.enable' wasn't found"}
Request Autofill.setAddresses failed. {"code":-32601,"message":"'Autofill.setAddresses' wasn't found"}
```

**原因**: 
- Electron 31.x 版本与 Chrome DevTools 的兼容性问题
- DevTools 尝试调用 Electron 不支持的 Autofill API
- 这是 Electron 的已知问题，不影响应用功能

**解决方案**:
- 方案 A: 禁用 DevTools 的 Autofill 功能（推荐）
- 方案 B: 忽略错误（不影响使用）
- 方案 C: 降级 Electron 版本（不推荐）

### 2. 内存占用分析

**主要内存占用点**:
1. **React 状态管理** (约 15-20MB) - 16 个 useState hooks
2. **日历计算** (约 10-15MB) - 重复计算和对象创建
3. **飞书同步管理器** (约 20-30MB) - API 缓存和状态
4. **事件监听器** (约 5-8MB) - 未清理的监听器
5. **日程数据存储** (约 5-10MB) - localStorage 操作

## 优化方案

### 阶段 1: 修复 Autofill 错误 + 快速内存优化 (预计降至 100-120MB)

#### 1.1 禁用 DevTools Autofill (优先级：高)
**文件**: `src/main/index.ts`

**实施**:
```typescript
// 在创建窗口时禁用某些 DevTools 功能
mainWindow.webContents.on('devtools-opened', () => {
  // 禁用 Autofill 相关功能，避免错误
  mainWindow.webContents.executeJavaScript(`
    console.log('Autofill disabled for development');
  `);
});
```

**或者**: 在 electron-builder 配置中禁用
```json
{
  "build": {
    "electronCompile": false
  }
}
```

#### 1.2 使用 useMemo 缓存日历计算
**文件**: `src/renderer/src/App.tsx`

**实施**:
```typescript
import { useMemo } from 'react'

// 缓存 calendarDays 计算
const calendarDays = useMemo(() => {
  return getCalendarDays(calendarBaseDate.getFullYear(), calendarBaseDate.getMonth())
}, [calendarBaseDate])

// 缓存 eventsByDate
const eventsByDate = useMemo(() => {
  const map = new Map<string, CalendarEvent[]>()
  events.forEach(event => {
    if (!map.has(event.date)) {
      map.set(event.date, [])
    }
    map.get(event.date)!.push(event)
  })
  // 排序
  map.forEach(eventsList => {
    eventsList.sort((a, b) => formatTimeForSort(a.time) - formatTimeForSort(b.time))
  })
  return map
}, [events])
```

**预期效果**: 减少 15-20MB

#### 1.3 清理 useEffect 监听器
**文件**: `src/renderer/src/App.tsx`

**实施**:
```typescript
useEffect(() => {
  const cleanupMove = window.api?.onWindowMoved(handleMove)
  const cleanupResize = window.api?.onWindowResized(handleResize)
  
  return () => {
    cleanupMove?.()
    cleanupResize?.()
  }
}, [handleMove, handleResize])
```

**预期效果**: 防止内存泄漏，减少 5-8MB

#### 1.4 节流鼠标移动监听
**文件**: `src/renderer/src/App.tsx`

**实施**:
```typescript
const handleMouseMove = useMemo(() => {
  let lastCall = 0
  return (e: React.MouseEvent<HTMLDivElement>) => {
    const now = Date.now()
    if (now - lastCall >= 100) {
      lastCall = now
      const rect = e.currentTarget.getBoundingClientRect()
      setMousePosition({
        x: e.clientX - rect.left,
        y: e.clientY - rect.top
      })
    }
  }
}, [])
```

**预期效果**: 减少 30-50% 的鼠标事件处理

### 阶段 2: 深度优化 (预计降至 80-100MB)

#### 2.1 React.memo 优化子组件
**文件**: SettingsModal, ExportModal, EventFormModal

**实施**:
```typescript
const SettingsModal = React.memo(({ isOpen, onClose, onSave, settings }) => {
  // 组件内容
})

// 使用 useMemo 包装 props
const settingsProps = useMemo(() => ({
  isOpen: settingsOpen,
  onClose: () => setSettingsOpen(false),
  onSave: handleSaveSettings,
  settings
}), [settingsOpen, settings])
```

**预期效果**: 减少 10-15MB

#### 2.2 懒加载模态框组件
**文件**: `src/renderer/src/App.tsx`

**实施**:
```typescript
import { lazy, Suspense } from 'react'

const SettingsModal = lazy(() => import('./SettingsModal'))
const ExportModal = lazy(() => import('./ExportModal'))
const EventFormModal = lazy(() => import('./EventFormModal'))

// 使用时
<Suspense fallback={null}>
  {settingsOpen && <SettingsModal />}
</Suspense>
```

**预期效果**: 减少 10-15MB 初始加载

#### 2.3 优化数据存储
**文件**: `src/renderer/src/App.tsx`

**实施**:
```typescript
// 防抖保存 localStorage
const saveEventsToLocalStorage = useMemo(() => {
  let timeoutId: NodeJS.Timeout | null = null
  return (eventsToSave: CalendarEvent[]) => {
    if (timeoutId) {
      clearTimeout(timeoutId)
    }
    timeoutId = setTimeout(() => {
      localStorage.setItem('calendar-events', JSON.stringify(eventsToSave))
    }, 300)
  }
}, [])
```

**预期效果**: 减少 5-8MB
