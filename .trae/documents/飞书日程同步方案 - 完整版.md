# 飞书日程双向同步方案（完整版）

## 📋 需求分析

### 当前状态
- ✅ 本地日历已实现创建、编辑、删除功能
- ✅ 本地数据持久化（localStorage）
- ✅ 飞书日程创建功能（单向）
- ✅ 飞书日程更新功能（单向）
- ❌ 飞书日程同步到本地（缺失）
- ❌ 双向同步冲突处理（缺失）
- ❌ 定时同步机制（缺失）

### 目标
实现本地日历与飞书日历的**双向同步**，确保：
1. 本地创建的日程自动同步到飞书
2. 飞书已有的日程自动同步到本地
3. 冲突时以最新修改为准
4. 支持增量同步，减少 API 调用

## 🔧 技术方案

### 一、飞书日历 API 详解

#### 1. 获取日程列表 API

**请求格式：**
```http
GET https://open.feishu.cn/open-apis/calendar/v4/calendars/{calendar_id}/events
Authorization: Bearer {tenant_access_token}
Content-Type: application/json
```

**查询参数（三种方式）：**

| 方式 | 参数 | 适用场景 | 特点 |
|------|------|----------|------|
| 时间范围 | `start_time` + `end_time` | 初次同步 | 一次性返回，无法分页 |
| 锚点查询 | `anchor_time` + `page_token` | 增量拉取 | 支持分页 |
| 增量同步 | `sync_token` | 定时同步 | 只返回变更数据 |

**响应关键字段：**
- `event_id`: 日程唯一 ID
- `summary`: 日程标题
- `start_time.timestamp`: 开始时间戳（**字符串格式**）
- `end_time.timestamp`: 结束时间戳（**字符串格式**）
- `updated_time`: 最后更新时间戳（秒），**用于冲突判断**
- `sync_token`: 增量同步标记

#### 2. 创建日程 API

**请求格式：**
```http
POST https://open.feishu.cn/open-apis/calendar/v4/calendars/{calendar_id}/events
Authorization: Bearer {tenant_access_token}
Content-Type: application/json
Locale: zh_cn
```

**请求体：**
```json
{
  "summary": "日程标题",
  "description": "日程描述",
  "start_time": {
    "timestamp": "1709712000",
    "timezone": "Asia/Shanghai"
  },
  "end_time": {
    "timestamp": "1709715600",
    "timezone": "Asia/Shanghai"
  },
  "location": "会议室 A",
  "need_notification": true
}
```

**响应：**
```json
{
  "code": 0,
  "data": {
    "event_id": "xxxxx",
    "summary": "日程标题",
    "updated_time": "1709700000"
  }
}
```

#### 3. 更新日程 API ⭐

**请求格式：**
```http
PATCH https://open.feishu.cn/open-apis/calendar/v4/calendars/{calendar_id}/events/{event_id}
Authorization: Bearer {tenant_access_token}
Content-Type: application/json
Locale: zh_cn
```

**路径参数：**
- `calendar_id`: 日历 ID
- `event_id`: 日程 ID（创建时返回）

**请求体（只需提供要修改的字段）：**
```json
{
  "summary": "修改后的标题",
  "start_time": {
    "timestamp": "1709715600",
    "timezone": "Asia/Shanghai"
  },
  "end_time": {
    "timestamp": "1709719200",
    "timezone": "Asia/Shanghai"
  },
  "location": "新的地点",
  "need_notification": false
}
```

**重要说明：**
- ✅ **只需提供要修改的字段**，不需要完整数据
- ✅ 不传值的字段表示不更新
- ✅ 当前身份为**组织者**时，可修改所有字段
- ⚠️ 当前身份为**参与者**时，仅可编辑部分字段

**响应：**
```json
{
  "code": 0,
  "data": {
    "event_id": "xxxxx",
    "updated_time": "1709720000"
  }
}
```

**错误码：**
- `193001`: 日程未找到
- `193002`: 无权限操作
- `193003`: 日程已经被删除

#### 4. 删除日程 API ⭐

**请求格式：**
```http
DELETE https://open.feishu.cn/open-apis/calendar/v4/calendars/{calendar_id}/events/{event_id}
Authorization: Bearer {tenant_access_token}
```

**路径参数：**
- `calendar_id`: 日历 ID
- `event_id`: 要删除的日程 ID

**响应：**
```json
{
  "code": 0,
  "msg": "success"
}
```

**注意事项：**
- ⚠️ 删除后无法恢复
- ⚠️ 必须有日历的 `writer` 或 `owner` 权限
- ⚠️ 重复日程需要提供完整的 event_id（包括例外日程后缀）

### 二、同步策略

#### 1. 数据模型映射

**飞书 → 本地：**
```typescript
interface FeishuEvent {
  event_id: string          // 日程 ID
  summary: string           // 标题
  description?: string      // 描述
  start_time: {
    timestamp: string       // 开始时间戳（秒）
    timezone: string        // 时区
  }
  end_time: {
    timestamp: string       // 结束时间戳（秒）
    timezone: string
  }
  location?: string         // 地点
  visibility?: string       // 公开范围
  updated_time: string      // 最后更新时间戳（秒）⭐
}

interface LocalEvent {
  id: string                // 本地 ID
  date: string              // YYYY-MM-DD
  time: string              // HH:mm
  title: string             // 标题
  location: string          // 地点
  description?: string      // 描述
  feishuEventId?: string    // 飞书日程 ID ⭐
  endTime?: string          // ISO 格式结束时间
  lastSyncTime?: number     // 最后同步时间戳 ⭐
}
```

**转换函数：**
```typescript
// 飞书时间戳 → 本地日期时间
function convertToLocalDate(timestamp: string): { date: string; time: string } {
  const date = new Date(parseInt(timestamp) * 1000)
  return {
    date: formatDate(date),  // YYYY-MM-DD
    time: formatTime(date)   // HH:mm
  }
}

// 本地日期时间 → 飞书时间戳
function toFeishuTimestamp(date: string, time: string): string {
  const timestamp = new Date(`${date}T${time}:00`).getTime()
  return Math.floor(timestamp / 1000).toString()
}
```

#### 2. 同步流程设计

##### 2.1 初次同步（全量同步）

**场景：** 用户首次使用同步功能

**步骤：**
```
1. 确定同步时间范围（前后 30 天）
2. 调用飞书 API（start_time + end_time）
3. 遍历飞书日程，转换为本地格式
4. 批量保存到本地（localStorage）
5. 记录 sync_token 和最后同步时间
```

**代码实现：**
```typescript
async function initialSync(): Promise<SyncResult> {
  const now = new Date()
  const startTime = new Date(now.getTime() - 30 * 24 * 3600000)
  const endTime = new Date(now.getTime() + 30 * 24 * 3600000)
  
  const result = await window.api.feishu.getEvents(
    FEISHU_CONFIG.testCalendarId,
    Math.floor(startTime.getTime() / 1000),
    Math.floor(endTime.getTime() / 1000)
  )
  
  if (!result.success) {
    throw new Error(result.error)
  }
  
  // 转换为本地格式
  const localEvents: CalendarEvent[] = result.events.map(feishuEvent => {
    const startTime = convertToLocalDate(feishuEvent.start_time.timestamp)
    const endTime = new Date(parseInt(feishuEvent.end_time.timestamp) * 1000)
    
    return {
      id: generateLocalId(),
      date: startTime.date,
      time: startTime.time,
      title: feishuEvent.summary,
      location: feishuEvent.location || '',
      description: feishuEvent.description || '',
      feishuEventId: feishuEvent.event_id,
      endTime: endTime.toISOString(),
      lastSyncTime: Date.now()
    }
  })
  
  // 合并到本地数据
  mergeEvents(localEvents)
  
  // 保存 sync_token
  saveSyncToken(result.sync_token)
  
  return {
    added: localEvents.length,
    updated: 0,
    deleted: 0
  }
}
```

##### 2.2 增量同步（定时同步）

**场景：** 定期同步飞书日程变更

**步骤：**
```
1. 从本地获取上次的 sync_token
2. 调用飞书 API（携带 sync_token）
3. 获取变更的日程列表
4. 对比本地数据，判断新增/更新/删除
5. 更新本地数据
6. 保存新的 sync_token
```

**代码实现：**
```typescript
async function incrementalSync(): Promise<SyncResult> {
  const syncToken = loadSyncToken()
  
  if (!syncToken) {
    // 没有 sync_token，执行初次同步
    return await initialSync()
  }
  
  // 使用 sync_token 增量获取
  const result = await window.api.feishu.getEventsWithSyncToken(
    FEISHU_CONFIG.testCalendarId,
    syncToken
  )
  
  if (!result.success) {
    throw new Error(result.error)
  }
  
  const stats = { added: 0, updated: 0, deleted: 0 }
  
  // 处理变更
  for (const feishuEvent of result.events) {
    const localEvent = findLocalByFeishuId(feishuEvent.event_id)
    
    if (!localEvent) {
      // 新增日程
      addLocalEvent(convertToLocalEvent(feishuEvent))
      stats.added++
    } else {
      // 判断冲突：以最后更新时间为准
      const feishuUpdateTime = parseInt(feishuEvent.updated_time) * 1000
      const localUpdateTime = localEvent.lastSyncTime || 0
      
      if (feishuUpdateTime > localUpdateTime) {
        // 飞书更新更晚，覆盖本地
        updateLocalEvent(convertToLocalEvent(feishuEvent))
        stats.updated++
      } else {
        // 本地更新更晚，同步到飞书
        await syncUpdateToFeishu(localEvent)
      }
    }
  }
  
  // 保存新的 sync_token
  saveSyncToken(result.sync_token)
  
  return stats
}
```

```typescript
async function syncCreateToFeishu(event: CalendarEvent): Promise<boolean> {
  try {
    const feishuEventData = {
      summary: event.title,
      description: event.description || '',
      start_time: {
        timestamp: toFeishuTimestamp(event.date, event.time),
        timezone: 'Asia/Shanghai'
      },
      end_time: {
        timestamp: Math.floor(new Date(event.endTime!).getTime() / 1000),
        timezone: 'Asia/Shanghai'
      },
      location: event.location,
      need_notification: false
    }
    
    const result = await window.api.feishu.createEvent(
      FEISHU_CONFIG.testCalendarId,
      feishuEventData
    )
    
    if (result.success) {
      // 更新本地记录飞书 event_id
      updateEventFeishuId(event.id, result.event.event_id)
      updateEventLastSyncTime(event.id, Date.now())
      return true
    }
    
    return false
  } catch (error) {
    console.error('Sync to Feishu failed:', error)
    return false
  }
}
```

##### 2.4 本地更新同步到飞书

**场景：** 用户在本地修改日程后

**步骤：**
```
1. 保存本地数据
2. 异步调用飞书更新 API（只传修改的字段）
3. 更新本地 lastSyncTime
```

**代码实现：**
```typescript
async function syncUpdateToFeishu(event: CalendarEvent): Promise<boolean> {
  try {
    if (!event.feishuEventId) {
      // 没有飞书 ID，执行创建
      return await syncCreateToFeishu(event)
    }
    
    const updateData: any = {}
    
    // 只包含修改的字段
    if (event.title) {
      updateData.summary = event.title
    }
    if (event.description) {
      updateData.description = event.description
    }
    if (event.date && event.time) {
      updateData.start_time = {
        timestamp: toFeishuTimestamp(event.date, event.time),
        timezone: 'Asia/Shanghai'
      }
    }
    if (event.endTime) {
      updateData.end_time = {
        timestamp: Math.floor(new Date(event.endTime).getTime() / 1000),
        timezone: 'Asia/Shanghai'
      }
    }
    if (event.location) {
      updateData.location = event.location
    }
    
    const result = await window.api.feishu.updateEvent(
      FEISHU_CONFIG.testCalendarId,
      event.feishuEventId,
      updateData
    )
    
    if (result.success) {
      updateEventLastSyncTime(event.id, Date.now())
      return true
    }
    
    return false
  } catch (error) {
    console.error('Update Feishu event failed:', error)
    return false
  }
}
```

#### 3. 冲突解决策略

**冲突场景：**
- 本地和飞书同时修改了同一个日程
- 网络延迟导致同步时序错乱

**解决方案：Last Write Wins（最后写入获胜）**

```typescript
function resolveConflict(
  localEvent: LocalEvent, 
  feishuEvent: FeishuEvent
): 'use_local' | 'use_feishu' {
  const localTime = localEvent.lastSyncTime || 0
  const feishuTime = parseInt(feishuEvent.updated_time) * 1000
  
  if (feishuTime > localTime) {
    // 飞书更新更晚，以飞书为准
    return 'use_feishu'
  } else if (localTime > feishuTime) {
    // 本地更新更晚，以本地为准
    return 'use_local'
  } else {
    // 时间相同，以飞书为准（避免重复）
    return 'use_feishu'
  }
}
```

#### 2. 核心类设计

**SyncManager（同步管理器）：**
```typescript
class SyncManager {
  private syncToken: string | null = null
  private lastSyncTime: number = 0
  private isSyncing: boolean = false
  
  // 初始化同步
  async initialize(): Promise<void> {
    this.loadSyncToken()
    this.loadLastSyncTime()
  }
  
  // 增量同步
  async sync(): Promise<SyncResult> {
    if (this.isSyncing) return { added: 0, updated: 0, deleted: 0 }
    
    this.isSyncing = true
    try {
      const result = await incrementalSync()
      this.lastSyncTime = Date.now()
      return result
    } finally {
      this.isSyncing = false
    }
  }
  
  // 全量同步
  async fullSync(timeRange: TimeRange): Promise<SyncResult> {
    return await initialSync()
  }
  
  // 同步单个日程到飞书
  async syncEventToFeishu(event: CalendarEvent, action: 'create' | 'update' | 'delete'): Promise<boolean> {
    switch (action) {
      case 'create':
        return await syncCreateToFeishu(event)
      case 'update':
        return await syncUpdateToFeishu(event)
      case 'delete':
        return await syncDeleteToFeishu(event)
    }
  }
  
  private loadSyncToken(): void {
    const token = localStorage.getItem('feishu_sync_token')
    this.syncToken = token
  }
  
  private saveSyncToken(token: string): void {
    this.syncToken = token
    localStorage.setItem('feishu_sync_token', token)
  }
}
```

**SyncWorker（后台同步）：**
```typescript
class SyncWorker {
  private interval: NodeJS.Timeout | null = null
  private syncManager: SyncManager
  
  constructor(syncManager: SyncManager) {
    this.syncManager = syncManager
  }
  
  // 启动定时同步
  start(intervalMinutes: number = 30): void {
    this.stop()
    this.interval = setInterval(async () => {
      await this.syncManager.sync()
    }, intervalMinutes * 60 * 1000)
  }
  
  // 停止定时同步
  stop(): void {
    if (this.interval) {
      clearInterval(this.interval)
      this.interval = null
    }
  }
  
  // 立即同步
  async syncNow(): Promise<SyncResult> {
    return await this.syncManager.sync()
  }
}
```
