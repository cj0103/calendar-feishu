# 导出可执行程序和开机自启动计划

## 一、目标

1. 打包 Electron 应用为可执行程序
2. 配置开机自启动功能
3. 在单位电脑上部署测试

---

## 二、导出可执行程序

### 2.1 检查打包配置

**查看文件：** `electron-builder.json`

**需要确认的配置：**
- 应用名称和版本
- 图标文件
- 输出目录
- Windows 特定配置（NSIS 或 portable）
- 文件关联
- 协议注册

### 2.2 打包命令

**构建并打包：**
```bash
npm run package
```

**或者分步执行：**
```bash
# 先构建
npm run build

# 再打包
npm run package
```

**预期输出：**
- `dist/` 目录
- `dist/win-unpacked/` - 未打包的版本（可用于测试）
- `dist/setup.exe` 或 `dist/*.exe` - 安装程序（如果配置了 NSIS）

### 2.3 打包选项

**可能的打包方式：**

1. **NSIS 安装包**（推荐用于生产环境）
   - 生成安装程序
   - 支持卸载
   - 支持开机自启动配置

2. **Portable 便携版**
   - 单文件可执行
   - 无需安装
   - 适合 U 盘携带

3. **Unpacked 解压版**
   - 直接解压即可运行
   - 适合测试
   - 文件较大

## 三、配置开机自启动

### 3.1 Electron 实现方案

**方案 A：使用 electron-builder 的 auto-launch 配置（推荐）**

**配置位置：** `electron-builder.json`

```json
{
  "win": {
    "target": "nsis",
    "requestedExecutionLevel": "asInvoker"
  },
  "nsis": {
    "oneClick": false,
    "allowToChangeInstallationDirectory": true,
    "createDesktopShortcut": true,
    "createStartMenuShortcut": true,
    "runAfterFinish": true
  }
}
```

**代码实现：** 在应用启动时检查并设置自启动

```typescript
import { app } from 'electron'

// 设置开机自启动
app.setLoginItemSettings({
  openAtLogin: true,  // 开机自启动
  openAsHidden: true,  // 隐藏启动（不显示窗口）
  name: '桌面日历'  // 启动项名称
})
```

---

### 方案 B：使用 electron-updater 的 auto-launch 模块

**安装依赖：**
```bash
npm install electron-updater
```

**代码实现：**
```typescript
import AutoLaunch from 'auto-launch'

const appLauncher = new AutoLaunch({
  name: '桌面日历',
  path: app.getPath('exe'),
  isHidden: true
})

// 启用自启动
appLauncher.enable()
```

---

### 方案 C：Windows 注册表（最底层）

**代码实现：**
```typescript
import { exec } from 'child_process'

const setAutoLaunch = () => {
  const exePath = app.getPath('exe')
  const command = `reg add "HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run" /v "DesktopCalendar" /t REG_SZ /f /d "${exePath}"`
  
  exec(command, (error) => {
    if (error) {
      console.error('设置自启动失败:', error)
    } else {
      console.log('设置自启动成功')
    }
  })
}
```

---

### 3.2 推荐方案

**使用方案 A + 设置界面**

**实现步骤：**

1. **在设置中添加开关**
   - 开机自启动 ☑️
   - 隐藏启动 ☑️

2. **代码实现**

### 5.2 设置界面添加

**在 SettingsModal.tsx 中添加：**

```tsx
// 开机自启动设置
<div className="setting-item">
  <label>
    <input
      type="checkbox"
      checked={autoLaunch}
      onChange={(e) => setAutoLaunch(e.target.checked)}
    />
    开机自启动

  {autoLaunch && (
    <label>
      <input
        type="checkbox"
```

### 5.3 主进程实现

**在 src/main/index.ts 中添加：**

```typescript
import { app } from 'electron'
import Store from 'electron-store'

const store = new Store()

app.whenReady().then(() => {
  // 读取配置
  const autoLaunch = store.get('autoLaunch', false)
  const launchHidden = store.get('launchHidden', false)
  
  // 设置自启动
  if (autoLaunch) {
    app.setLoginItemSettings({
      openAtLogin: true,
      openAsHidden: launchHidden,
      name: '桌面日历'
    })
  }
  
  // 监听设置变化
  ipcMain.on('set-auto-launch', (_, value) => {
    store.set('autoLaunch', value)
    app.setLoginItemSettings({
      openAtLogin: value,
      openAsHidden: store.get('launchHidden', false),
      name: '桌面日历'
    })
  })
  
  ipcMain.on('set-launch-hidden', (_, value) => {
    store.set('launchHidden', value)
    app.setLoginItemSettings({
      openAtLogin: store.get('autoLaunch', false),
      openAsHidden: value,
      name: '桌面日历'
    })
  })
})
```

## 七、可能的问题和解决方案

### 问题 1：打包失败

**可能原因：**
- 缺少依赖
- 配置错误
- 图标文件缺失

**解决方案：**
```bash
# 清理并重新安装
rm -rf node_modules package-lock.json
npm install

# 清理构建缓存
npm run build

# 重新打包
npm run package
```

---

### 问题 2：自启动不工作

**可能原因：**
- Windows 权限问题
- 路径包含中文
- 杀毒软件阻止

**解决方案：**
- 以管理员身份运行一次
- 检查 Windows 启动文件夹
- 查看杀毒软件日志

---

### 问题 3：打包后无法运行

**可能原因：**
- 缺少运行时依赖
- 路径问题
- 权限问题

**解决方案：**
- 检查 `electron-builder.json` 配置
- 查看应用日志
- 尝试 portable 版本
