# 简化同步逻辑方案

## 🎯 核心思路

**简化原则**：
1. **飞书 → 本地**：一次性全量导入，后续按需更新
2. **本地 → 飞书**：只同步本地独有的数据（没有 `feishuEventId`）
3. **队列管理**：只在本地操作时添加队列，不重复处理已同步的数据

## 📋 实施方案

### 功能 1：飞书全量导入按钮

**位置**：飞书日历管理中心页面

**按钮名称**：`从飞书同步` 或 `全量下载`

**功能**：
- 点击后从飞书全量下载该日历的所有日程
- 覆盖本地数据（或智能合并）
- 清空该日历的同步队列（因为已全量同步）

**实现**：
```typescript
// 在飞书管理页面添加按钮

  📥 从飞书同步
</button>

    // 获取所有日程

    // 转换为本地格式

```

### 功能 2：优化队列管理

**启动时**：
```typescript
// 只添加本地独有的数据到队列（没有 feishuEventId）
const localEvents = loadEventsFromLocalStorage()
const localOnlyEvents = localEvents.filter(e => !e.feishuEventId)

for (const event of localOnlyEvents) {
  syncQueue.add(event, 'create')

```

**本地操作时**：
```typescript
// 新增日程
const handleAddEvent = (event: CalendarEvent) => {
  // 添加到本地
  addLocalEvent(event)
  // 如果没有 feishuEventId，添加到队列
  if (!event.feishuEventId) {
    syncQueue.add(event, 'create')
  }
}

// 编辑日程
const handleEditEvent = (event: CalendarEvent) => {
  // 更新本地
  updateLocalEvent(event)
  // 如果有 feishuEventId，添加到更新队列
  if (event.feishuEventId) {
    syncQueue.add(event, 'update')
  }
}

// 删除日程
const handleDeleteEvent = (event: CalendarEvent) => {
  // 从本地删除
  deleteLocalEvent(event)
  // 如果有 feishuEventId，添加到删除队列
  if (event.feishuEventId) {
    syncQueue.add(event, 'delete')
  }
}
```

### 功能 3：简化同步逻辑

**sync() 方法简化**：
```typescript
async sync(): Promise<SyncResult> {
  // 不再从飞书增量同步
  // 只处理队列中的本地 → 飞书同步
  
  const stats = { added: 0, updated: 0, deleted: 0, uploaded: 0 }
  
  // 队列会自动处理，这里只返回统计
  const queueStats = syncQueue.getStats()
  stats.uploaded = queueStats.total
  
  return stats
}
```

**移除 incrementalSync**：
- 不再每次同步都从飞书获取数据
- 不再检测本地独有的事件（启动时已处理）
- 不再重复添加已同步的事件到队列

## 🔄 工作流程

### 场景 1：首次使用

1. 用户点击"从飞书同步"
2. 全量下载飞书日程到本地
3. 所有日程都有 `feishuEventId`
4. 队列为空

### 场景 2：新增本地日程

1. 用户新增日程（没有 `feishuEventId`）
2. 自动添加到队列（create 类型）
3. 队列后台处理，上传到飞书
4. 获取 `feishuEventId`，更新本地

### 场景 3：编辑本地日程

1. 用户编辑日程（已有 `feishuEventId`）
2. 自动添加到队列（update 类型）
3. 队列后台处理，更新到飞书

### 场景 4：删除本地日程

1. 用户删除日程（已有 `feishuEventId`）
2. 自动添加到队列（delete 类型）
3. 队列后台处理，从飞书删除

### 场景 5：飞书端有变更

1. 用户点击"从飞书同步"
2. 全量下载飞书日程
3. 覆盖本地数据
4. 清空队列

```typescript
async sync(): Promise<SyncResult> {
  if (this.isSyncing) {
    return { added: 0, updated: 0, deleted: 0, uploaded: 0 }
  }
  
  this.isSyncing = true
  
  try {
    // 只处理队列，不再从飞书同步
    const queueStats = syncQueue.getStats()
    
    console.log(`📝 当前队列：${queueStats.total} 个任务`)
    
    // 队列会自动处理，这里只返回统计
    return {
      added: 0,
      updated: 0,
      deleted: 0,
      uploaded: queueStats.create + queueStats.update
    }
  } finally {
    this.isSyncing = false
  }
}
```

## ⚠️ 注意事项

1. **飞书端变更**：需要用户主动点击"从飞书同步"才能获取
2. **数据覆盖**：全量同步会覆盖本地数据，需要提示用户
3. **队列清理**：全量同步后需要清空对应日历的队列
