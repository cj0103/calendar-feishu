# 桌面常驻显示方案和功能优化计划

## 一、桌面常驻显示需求分析

### 当前状态
- ✅ 窗口固定桌面：不受最小化按钮影响
- ❌ 层级问题：需要实现在壁纸上层、其他应用下层

### 技术难点
Electron 窗口默认行为：
1. 普通窗口：在其他应用上层（不符合需求）
2. 置顶窗口：始终在最前（不符合需求）
3. 需要实现：桌面壁纸层和普通应用层之间

## 二、桌面常驻显示技术方案

### 方案对比

#### 方案 A：使用 setAlwaysOnBottom + 工具窗口（推荐）

**实现方式：**
```typescript
// 在主进程 index.ts 的 createWindow 中
mainWindow.setAlwaysOnBottom(true);  // 设置窗口在最底层
mainWindow.setSkipTaskbar(true);      // 不在任务栏显示
mainWindow.setVisibleOnAllWorkspaces(true, { visibleOnFullScreen: true });

// 设置窗口为工具窗口（不在 Alt+Tab 中显示）
mainWindow.setType('toolbar');  // 或 'dock'、'desktop'
```

**优点：**
- 实现简单，Electron 原生支持
- 性能开销小
- 兼容性好

**缺点：**
- 可能被某些全屏应用遮挡
- Windows 11 某些版本可能需要额外处理

**实现步骤：**
1. 在 `src/main/index.ts` 的 `createWindow` 函数中添加设置
2. 测试在不同场景下的显示层级
3. 添加窗口设置选项，让用户可选择是否常驻桌面

---

#### 方案 B：使用窗口层级设置（备选）

**实现方式：**
```typescript
mainWindow.setType('desktop');  // 设置为桌面层级
mainWindow.setAlwaysOnBottom(true);
```

**说明：**
- `setType('desktop')` 在某些系统上可以将窗口设置为桌面层级
- 需要测试 Windows 10/11 的兼容性

---

#### 方案 C：结合 Windows API（高级方案）

**实现方式：**
- 使用 Electron 的 `nativeImage` 和 `BrowserView`
- 通过 Windows API 设置窗口所有者为桌面窗口
- 需要额外的 Node.js 原生模块

**优点：**
- 最精确的层级控制
- 可以实现真正的桌面壁纸效果

**缺点：**
- 实现复杂
- 需要原生模块，增加打包体积
- 维护成本高

---

### 推荐实施方案：方案 A + 用户可选配置

**实施步骤：**

1. **修改窗口创建代码**（`src/main/index.ts`）
   ```typescript
   // 在 createWindow 函数中
   mainWindow.setAlwaysOnBottom(true);
   mainWindow.setSkipTaskbar(true);
   mainWindow.setType('toolbar');  // 或 'dock'
   ```

2. **添加用户配置选项**（`SettingsModal.tsx`）
   - [ ] 常驻桌面模式（开/关）
   - [ ] 任务栏显示（开/关）
   - [ ] 窗口类型选择（工具窗口/普通窗口）

3. **添加窗口控制功能**
   - 在设置中提供"恢复窗口位置"按钮
   - 防止用户找不到窗口

4. **测试验证**
   - 打开多个应用测试层级关系
   - 测试最小化/恢复行为
   - 测试全屏应用时的表现

### 3.2 第一优先级功能（后续开发）

**1. 日程快速筛选/搜索** ⭐⭐⭐⭐
- **需求**：快速找到特定日程
- **实现**：
  - 按优先级筛选
  - 按关键词搜索
  - 按日期范围筛选
- **UI**：在日历上方添加搜索框和筛选器

**2. 日程导出功能增强** ⭐⭐⭐⭐
- **需求**：导出日程到本地文件
- **实现**：
  - 导出为 Markdown（适合笔记软件）
  - 导出为 CSV（适合 Excel）
  - 导出为 JSON（数据备份）
  - 导出为 ICS（日历标准格式，可导入其他日历）
- **范围选择**：导出全部/导出指定日期范围

**3. 重复日程功能** ⭐⭐⭐⭐
- **需求**：周期性会议（如每周一例会）
- **实现**：
  - 每天重复
  - 每周重复（可指定周几）
  - 每月重复（可指定日期）
  - 自定义重复规则
- **结束条件**：永不结束/指定结束日期/重复次数

**6. 字体和布局自定义** ⭐⭐⭐
- **需求**：个性化显示效果
- **实现**：

  - 字号调整（已实现日历字号，可扩展到全部）

  - 网格密度调整

**9. 快捷操作/快捷键** ⭐⭐⭐
- **需求**：提高效率
- **实现**：
  - 全局快捷键快速添加日程

- **实现**：
