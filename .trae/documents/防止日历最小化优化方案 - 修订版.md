# 防止日历最小化优化方案 - 修订版

## 问题分析

### 当前实现的问题
- ❌ **被动恢复**：日历先被最小化，然后在其他事件触发后才恢复
- ❌ **用户体验差**：用户会看到日历先消失再出现
- ❌ **闪烁问题**：最小化和恢复之间有时间差

### 根本原因
当前的 `minimize` 事件监听是在最小化**发生后**才触发，属于事后补救，而不是事前预防。

---

## 更优解决方案

### 方案 1：完全禁用最小化（推荐）⭐⭐⭐⭐⭐

**核心思路：**
不在最小化后才阻止，而是从根本上让窗口无法被最小化。

**实现方式：**

```typescript
// 在创建窗口时设置
mainWindow = new BrowserWindow({
  // ...其他配置
  minimizable: false,      // 禁用最小化按钮
  skipTaskbar: true,       // 不在任务栏显示
})

// 关键：监听系统级的"显示桌面"命令
// Windows 的"显示桌面"会触发所有窗口最小化
// 我们需要在这个动作发生前拦截
```

**问题：** Electron 没有提供直接拦截"显示桌面"事件的 API

**变通方案：**
```typescript
// 持续监控窗口状态，发现被最小化立即恢复
const preventMinimize = () => {
  if (mainWindow && mainWindow.isMinimized()) {
    mainWindow.restore()
    mainWindow.setAlwaysOnBottom(true)
  }
  setTimeout(preventMinimize, 100)  // 每 100ms 检查一次
}

// 启动监控
preventMinimize()
```

**优点：**
- 实现简单
- 响应快速（100ms 延迟）
- 不依赖系统 API

**缺点：**
- 轮询有轻微性能开销
- 仍会有极短暂的最小化状态

---

### 方案 2：使用 Windows 特定 API（最优但复杂）⭐⭐⭐⭐

**核心思路：**
通过 Windows API 设置窗口的扩展样式，让系统忽略"显示桌面"命令。

**实现方式：**
```typescript
import { nativeImage } from 'electron'

// 使用 Windows API 设置窗口所有者为桌面
// 这样"显示桌面"命令就不会影响我们的窗口
const setWindowAsDesktop = (window: BrowserWindow) => {
  const hwnd = window.getNativeWindowHandle()
  // 调用 Windows API SetWindowLong
  // 设置 GWL_EXSTYLE 为 WS_EX_TOOLWINDOW | WS_EX_LAYERED
  // 这样窗口就不会被"显示桌面"最小化
}
```

**需要依赖：**
- `node-ffi-napi` 或 `ref-napi` 调用 Windows API
- 或使用预编译的 native 模块

**优点：**
- 系统级解决方案
- 完全避免最小化
- 性能最优

**缺点：**
- 需要 native 模块
- 增加打包体积
- 只支持 Windows

### 方案 3：修改窗口类型（折中方案）⭐⭐⭐

**核心思路：**
使用不同的窗口类型，某些类型不会被"显示桌面"影响。

**实现方式：**
```typescript
mainWindow = new BrowserWindow({
  // ...其他配置
  type: 'desktop',  // 或 'panel'、'floating'
})
```

**窗口类型说明：**
- `desktop` - 桌面窗口（可能被系统限制）
- `panel` - 面板窗口
- `floating` - 浮动窗口
- `toolbar` - 工具窗口（当前使用）

**测试结果：**
- `toolbar` 类型在某些 Windows 版本上仍会被最小化
- `desktop` 类型可能不被支持
- 需要实际测试不同系统

### 方案 4：结合多种方法（实用方案）⭐⭐⭐⭐

**核心思路：**
结合方案 1 的轮询 + 事件监听，实现快速恢复。

**实现方式：**

```typescript
// 1. 禁用最小化按钮
mainWindow.minimizable = false

// 2. 监听 minimize 事件（第一道防线）
mainWindow.on('minimize', (event) => {
  event.preventDefault()
  setTimeout(() => {
    mainWindow.restore()
    mainWindow.show()
    mainWindow.setAlwaysOnBottom(true)
  }, 10)  // 缩短延迟到 10ms
})

// 3. 轮询检查（第二道防线）
const checkWindowState = () => {
  if (!mainWindow) return
  
  // 如果窗口被最小化但未触发 minimize 事件
  if (mainWindow.isMinimized()) {
    console.log('Detected minimized state via polling, restoring...')
    mainWindow.restore()
    mainWindow.show()
    mainWindow.setAlwaysOnBottom(true)
  }
  
  // 如果窗口不可见但不是最小化
  if (!mainWindow.isVisible() && !mainWindow.isMinimized()) {
    console.log('Window not visible, showing...')
    mainWindow.show()
    mainWindow.setAlwaysOnBottom(true)
  }
  
  setTimeout(checkWindowState, 200)  // 200ms 检查一次
}

// 启动轮询
checkWindowState()

// 4. 监听应用失去焦点（确保层级）
app.on('browser-window-focus', () => {
  // 当其他窗口获得焦点时，确保日历在最底层
  setTimeout(() => {
    mainWindow?.setAlwaysOnBottom(true)
  }, 50)
})
```

**优点：**
- 多层防护
- 响应快速
- 不依赖 native 模块
- 兼容性好

**缺点：**
- 有轮询开销（但很小）
- 代码稍复杂

```typescript
mainWindow.on('minimize', (event) => {

      mainWindow.show()
      mainWindow.setAlwaysOnBottom(true)
    }
  })
})
```

```typescript
// 在 createWindow 函数末尾添加
const preventMinimizePolling = () => {
  if (!mainWindow) return
  
  try {
    if (mainWindow.isMinimized()) {
      console.log('[Polling] Detected minimized, restoring...')
      mainWindow.restore()
      mainWindow.show()
      mainWindow.setAlwaysOnBottom(true)
    }
  } catch (error) {
    console.error('[Polling] Error checking window state:', error)
  }
  
  setTimeout(preventMinimizePolling, 200)
}

preventMinimizePolling()
```

```typescript
  if (focusedWindow !== mainWindow) {
    setTimeout(() => {
      mainWindow?.setAlwaysOnBottom(true)
    }, 50)
  }
})
```

## 注意事项

### 1. 轮询间隔
- 太短（<50ms）：性能开销大
- 太长（>500ms）：恢复延迟明显
- **推荐**：200ms（平衡性能和响应）

### 2. 性能优化
```typescript
// 只在窗口可见时轮询
const preventMinimizePolling = () => {
  if (!mainWindow || !mainWindow.isVisible()) {
    setTimeout(preventMinimizePolling, 1000)  // 不可见时降低频率
    return
  }
  // ...正常检查
  setTimeout(preventMinimizePolling, 200)
}
```

### 3. 内存泄漏预防
```typescript
// 在窗口关闭时清理
mainWindow.on('closed', () => {
  mainWindow = null
  // 轮询会自动停止（检查 !mainWindow）
})
```
