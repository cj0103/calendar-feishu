## 数据格式说明

### JSON 结构
```json
{
  "Name": "ShuYZ 中国节假日",
  "Version": "1.0",
  "Generated": "生成时间",
  "Timezone": "Asia/Shanghai",
  "Author": "ShuYZ",
  "URL": "数据源地址",
  "Years": {
    "2024": [
      {
        "Name": "春节",
        "StartDate": "2024-02-10",
        "EndDate": "2024-02-17",
        "Duration": 8,
        "CompDays": ["2024-02-04", "2024-02-18"],
        "Memo": "说明文字",
        "URL": "政府文件链接"
      }
    ]
  }
}
```

### 字段含义

#### 顶层字段
- **Name**: 数据名称
- **Version**: 数据版本号
- **Generated**: 数据生成时间
- **Timezone**: 时区（Asia/Shanghai）
- **Author**: 数据作者
- **URL**: 数据源地址
- **Years**: 各年份的节假日数据（对象，键为年份）

#### 节假日信息字段
- **Name**: 节日名称（如"春节"、"国庆节"）
- **StartDate**: 放假开始日期（格式：YYYY-MM-DD）
- **EndDate**: 放假结束日期（格式：YYYY-MM-DD）
- **Duration**: 放假总天数
- **CompDays** (Compensatory Days): 调休工作日列表
  - 这些日期需要上班补班
  - 通常是放假日期前后的周末
  - 格式：["YYYY-MM-DD", "YYYY-MM-DD", ...]
- **Memo** (可选): 相关说明文字
- **URL** (可选): 政府文件链接

## 数据使用逻辑

### 节假日判断
1. **法定节假日** = `StartDate` 到 `EndDate` 之间的所有日期
   - 显示标记："休"
   - 背景色：与周末背景一致
   - 字体颜色：红色

2. **调休工作日** = `CompDays` 数组中的所有日期
   - 显示标记："班"
   - 背景色：与工作日背景一致
   - 字体颜色：绿色

3. **优先级**：调休工作日 > 法定节假日 > 自然周末 > 工作日

### 示例解析

#### 2024 年春节
```json
{
  "Name": "春节",
  "StartDate": "2024-02-10",
  "EndDate": "2024-02-17",
  "Duration": 8,
  "CompDays": ["2024-02-04", "2024-02-18"]
}
```

**含义**：
- **放假日期**：2024-02-10 至 2024-02-17（共 8 天）
  - 这些日期显示"休"字标记
- **调休上班**：2024-02-04（周日）、2024-02-18（周日）
  - 这两个日期显示"班"字标记
  - 虽然是周末，但需要上班

#### 2024 年国庆节
```json
{
  "Name": "国庆节",
  "StartDate": "2024-10-01",
  "EndDate": "2024-10-07",
  "Duration": 7,
  "CompDays": ["2024-09-29", "2024-10-12"]
}
```

**含义**：
- **放假日期**：2024-10-01 至 2024-10-07（共 7 天）
- **调休上班**：2024-09-29（周日）、2024-10-12（周六）

## 数据更新

### 更新频率
- **每年更新一次**
- 通常在每年年底前更新下一年的数据
- 中国政府一般在每年 11-12 月发布次年的节假日安排

### 更新步骤
1. 从 GitHub 下载最新的 `holidayAPI.json`
2. 替换项目中的 `src/renderer/public/holidayAPI.json`
3. 重启应用即可

### 更新注意事项
- 更新前建议备份旧数据
- 确保 JSON 格式正确
- 更新后测试各年份的节假日显示是否正常

## 常见问题

### Q: 为什么有些节假日没有调休日？
A: 不是所有节假日都需要调休。例如清明节、劳动节等短期假期通常不需要调休。

### Q: 数据不准确怎么办？
A: 请检查 JSON 文件是否为最新版本，或从官方渠道核实当年的节假日安排。

### Q: 可以手动修改节假日数据吗？
A: 可以，但建议保持 JSON 格式正确。修改后需要重启应用才能生效。
