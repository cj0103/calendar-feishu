# 防止日历窗口最小化优化计划

## 问题描述

当前问题：
1. 点击桌面最小化按钮时，日历窗口也会被最小化
2. 可能还有其他情况导致日历窗口不可见

## 问题分析

### 当前窗口配置
- ✅ `alwaysOnBottom: true` - 窗口在最底层
- ✅ `type: 'toolbar'` - 工具窗口类型
- ✅ `skipTaskbar: true` - 不在任务栏显示
- ❌ **缺少** - 防止最小化的处理

### 可能导致日历不可见的场景

1. **显示桌面按钮**（右下角小长条/Win+D）
   - 用户点击后所有窗口最小化
   - 日历也会被最小化

2. **任务栏右键菜单"显示桌面"**
   - 同上效果

3. **其他全屏应用**
   - 某些全屏应用可能会覆盖工具窗口

4. **虚拟桌面切换**
   - 切换到其他虚拟桌面时

## 解决方案

### 方案 A：监听最小化事件并立即恢复（推荐）

**实现思路：**
```typescript
// 监听窗口最小化事件
mainWindow.on('minimize', (event) => {
  // 阻止最小化
  event.preventDefault()
  // 立即恢复窗口
  mainWindow.restore()
  mainWindow.show()
})
```

**优点：**
- 实现简单
- 响应及时
- 用户体验好

**缺点：**
- 可能会有闪烁（极小概率）

---

### 方案 B：禁用最小化功能

**实现思路：**
```typescript
// 在创建窗口时设置
mainWindow = new BrowserWindow({
  // ...其他配置
  minimizable: false,  // 禁用最小化按钮
})
```

**注意：**
- 由于应用使用自定义无边框窗口，没有标准的最小化按钮
- 但系统级别的"显示桌面"仍会触发最小化
- 所以方案 B 需要配合方案 A 使用

---

### 方案 C：监听应用级别的最小化事件

**实现思路：**
```typescript
// 监听所有窗口的最小化
app.on('browser-window-minimize', (event, window) => {
  if (window === mainWindow) {
    // 取消最小化
    window.restore()
    window.show()
  }
})
```

---

### 综合方案（推荐）

结合以上方案：

1. **设置窗口不可最小化**
   ```typescript
   mainWindow.minimizable = false
   ```

2. **监听并阻止最小化事件**
   ```typescript
   mainWindow.on('minimize', (event) => {
     event.preventDefault()
     setTimeout(() => {
       mainWindow.restore()
       mainWindow.show()
       mainWindow.setAlwaysOnBottom(true)
     }, 50)
   })
   ```

3. **监听应用级最小化**
   ```typescript
   app.on('browser-window-minimize', (event, window) => {
     if (window === mainWindow) {
       event.preventDefault()
       setTimeout(() => {
         window.restore()
         window.show()
         window.setAlwaysOnBottom(true)
       }, 50)
     }
   })
   ```

4. **窗口显示时确保层级正确**
   ```typescript
   mainWindow.on('show', () => {
     mainWindow.setAlwaysOnBottom(true)
   })
   ```

## 实施步骤

### 1. 修改 `src/main/index.ts`

在 `createWindow` 函数中添加：

```typescript
// 设置窗口不可最小化
mainWindow.minimizable = false

// 监听窗口最小化事件
mainWindow.on('minimize', (event) => {
  console.log('Window minimize event detected, preventing...')
  event.preventDefault()
  
  // 延迟恢复，确保事件处理完成
  setTimeout(() => {
    if (mainWindow) {
      mainWindow.restore()
      mainWindow.show()
      mainWindow.setAlwaysOnBottom(true)
      console.log('Window restored and set to always on bottom')
    }
  }, 50)
})

// 监听应用级别的窗口最小化事件
app.on('browser-window-minimize', (event, window) => {
  if (window === mainWindow) {
    console.log('App-level minimize event detected, preventing...')
    event.preventDefault()
    
    setTimeout(() => {
      if (mainWindow) {
        window.restore()
        window.show()
        window.setAlwaysOnBottom(true)
        console.log('Window restored at app level')
      }
    }, 50)
  }
})
```

### 2. 增强现有事件处理

在现有的 `ready-to-show`、`show`、`blur` 事件中确保层级设置：

```typescript
// 已存在的 ready-to-show 事件
mainWindow.on('ready-to-show', () => {
  mainWindow.show()
  mainWindow.focus()
  mainWindow.setAlwaysOnBottom(true)  // 新增
})

// 已存在的 show 事件
mainWindow.on('show', () => {
  mainWindow.setVisibleOnAllWorkspaces(true, { visibleOnFullScreen: true })
  mainWindow.setAlwaysOnBottom(true)  // 新增
})
```

## 注意事项

### 1. 事件循环问题
- 使用 `setTimeout` 延迟恢复，避免事件冲突
- 延迟时间 50-100ms 为宜

### 2. 性能考虑
- 事件监听器只创建一次
- 避免频繁的 show/hide 切换

### 3. 用户体验
- 确保托盘菜单的"显示/隐藏"功能正常工作
- 不要阻止用户主动隐藏窗口（通过托盘）

### 4. 日志记录
- 添加详细的 console.log 用于调试
- 记录所有最小化相关事件
