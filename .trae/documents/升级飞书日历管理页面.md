# 升级飞书日历管理页面

## 当前问题
1. 页面名称"飞书日程测试页面"不够准确，该页面功能不仅是测试
2. 缺少日历管理功能（查看账号下所有日历、创建新日历）
3. 对于新安装用户，无法方便地创建专属日历

## 目标
将"飞书日程测试页面"升级为"飞书日历管理中心"，增加日历管理功能

## API 调研结果

### 1. 获取日历列表
- **接口**: `GET /calendar/v4/calendars`
- **说明**: 获取当前身份（应用/用户）可访问的日历列表
- **权限**: 需要对日历有 reader、writer 或 owner 权限
- **返回**: 
  ```json
  {
    "code": 0,
    "msg": "success",
    "data": {
      "items": [
        {
          "calendar_id": "feishu.cn_xxx@group.calendar.feishu.cn",
          "summary": "日历名称",
          "description": "日历描述",
          "permissions": "private",
          "color": -1,
          "type": "shared",
          "is_deleted": false,
          "is_third_party": false,
          "role": "owner"
        }
      ]
    }
  }
  ```

### 2. 创建共享日历
- **接口**: `POST /calendar/v4/calendars`
- **说明**: 为当前身份（应用/用户）创建一个共享日历
- **请求参数**:
  ```json
  {
    "summary": "测试日历",           // 日历标题（必填，最大 255 字符）
    "description": "使用开放接口创建日历",  // 日历描述（必填，最大 255 字符）
    "permissions": "private",        // 日历公开范围（private/show_only_free_busy/public）
    "color": -1,                     // 日历颜色（RGB 的 int32 表示）
    "summary_alias": "日历备注名"     // 日历备注名（最大 255 字符）
  }
  ```
- **返回**:
  ```json
  {
    "code": 0,
    "msg": "success",
    "data": {
      "calendar": {
        "calendar_id": "feishu.cn_xxxxxxxxxx@group.calendar.feishu.cn",
        "summary": "测试日历",
        "description": "使用开放接口创建日历",
        "permissions": "private",
        "color": -1,
        "type": "shared",
        "summary_alias": "日历备注名",
        "is_deleted": false,
        "is_third_party": false,
        "role": "owner"
      }
    }
  }
  ```

## 实现方案

### 1. 页面重命名
- **旧名称**: 飞书日程测试页面
- **新名称**: 飞书日历管理中心
- **文件名**: `FeishuCalendarManager.tsx`（可选，保持原名也可以）

### 2. 新增功能模块

#### 2.1 日历列表管理
- 显示账号下所有可访问的日历
- 显示信息：日历名称、日历 ID、权限角色、类型、颜色
- 支持选择某个日历作为操作目标
- 标记当前正在使用的日历（从 feishuConfig.ts 读取）

#### 2.2 创建新日历
- 表单输入：
  - 日历名称（必填）
  - 日历描述（可选）
  - 公开范围选择（private/show_only_free_busy/public）
  - 颜色选择（可选）
- 创建成功后：
  - 自动刷新日历列表
  - 提示用户是否更新 feishuConfig.ts 配置文件
  - 提供一键复制 calendarId 功能

#### 2.3 日历操作
- 复制日历 ID 到剪贴板
- 切换当前使用的日历（更新 feishuConfig.ts）
- 查看日历详情

### 3. 后端 API 扩展

在 `src/main/feishuCalendar.ts` 中已有 `getCalendarList()` 方法，需要新增：

```typescript
/**
 * 创建共享日历
 * @param summary 日历标题
 * @param description 日历描述
 * @param permissions 日历公开范围
 * @param color 日历颜色
 * @param summaryAlias 日历备注名
 */
async createCalendar(
  summary: string,
  description?: string,
  permissions?: 'private' | 'show_only_free_busy' | 'public',
  color?: number,
  summaryAlias?: string
): Promise<any>
```

在 `src/main/index.ts` 中新增 IPC 处理器：
```typescript
ipcMain.handle('feishu:createCalendar', async (event, config) => {
  return await feishuCalendarAPI.createCalendar(
    config.summary,
    config.description,
    config.permissions,
    config.color,
    config.summaryAlias
  )
})

ipcMain.handle('feishu:getCalendarList', async () => {
  return await feishuCalendarAPI.getCalendarList()
})
```

### 4. 前端 UI 设计

#### 4.1 布局结构
```
┌─────────────────────────────────────────┐
│  飞书日历管理中心                        │
├─────────────────────────────────────────┤
│  ┌─────────────────┐  ┌───────────────┐ │
│  │ 日历列表        │  │ 创建新日历    │ │
│  │ - 日历 1 (当前) │  │ 表单输入      │ │
│  │ - 日历 2        │  │               │ │
│  │ - 日历 3        │  │               │ │
│  └─────────────────┘  └───────────────┘ │
├─────────────────────────────────────────┤
│  日程获取与同步（原有功能）              │
│  - 日历 ID 选择                          │
│  - 时间范围选择                          │
│  - 获取日程 / 同步到本地                 │
└─────────────────────────────────────────┘
```

#### 4.2 交互流程
1. 用户打开页面，自动加载日历列表
2. 显示当前配置的 calendarId 对应的日历
3. 用户可以选择：
   - 从列表中选择其他日历 → 自动更新日历 ID 输入框
   - 创建新日历 → 填写表单 → 创建成功 → 提示更新配置
4. 原有日程获取/同步功能保持不变，但日历 ID 可以从列表选择

### 5. 配置管理

#### 方案 A：手动更新（推荐）
- 创建成功后显示新 calendarId
- 提供"复制"按钮
- 用户手动更新 `feishuConfig.ts`

#### 方案 B：自动更新（复杂，不推荐）
- 直接修改 `feishuConfig.ts` 文件
- 需要文件系统写权限
- 需要重启应用生效

**建议**: 采用方案 A，更安全透明
