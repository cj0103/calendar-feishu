# 飞书配置管理功能计划（持久化配置 - 最终方案）

## 🎯 任务目标

1. **持久化配置** - 一次配置，永久使用（保存到用户数据目录）
2. **不打包配置** - release 中不含 App ID/Secret（feishuConfig.ts 只保留示例值）
3. **首次启动引导** - 打开 release 时提示配置飞书凭证
4. **开发友好** - 开发时可以使用 feishuConfig.ts 的示例配置

## 📋 实施方案

### 核心思路

**配置文件**：

* `feishuConfig.ts` - 示例配置（打包到 release，但是示例值）

* `feishu-user-config.json` - 用户配置（不打包，保存到用户数据目录）

**配置读取逻辑**：

```
1. 应用启动
   ↓
2. 检查 feishu-user-config.json 是否存在
   ↓
   ├─ 存在 → 使用用户配置（已配置过）
   ↓
   └─ 不存在 → 使用 feishuConfig.ts 示例配置
       ↓
   提示用户配置（首次启动）
```

**打包策略**：

* `feishuConfig.ts` → 编译到 index.js（但只有示例值）

* `feishu-user-config.json` → 不打包（用户数据目录）

### 工作流程

#### 开发阶段

```
1. 开发者修改 feishuConfig.ts（填入自己的配置）
2. 运行 npm run dev
3. 应用使用 feishuConfig.ts 的配置
```

#### 用户阶段（release）

```
1. 用户首次打开 release
2. 检测到没有用户配置
3. 弹出配置向导（必须配置）
4. 用户输入 App ID、App Secret、Calendar ID
5. 保存到 feishu-user-config.json
6. 后续启动自动加载用户配置
```

### 2. 创建配置管理器

```typescript
// src/main/configManager.ts
import { app } from 'electron'
import fs from 'fs'
import path from 'path'

interface FeishuUserConfig {
  appId: string
  appSecret: string
  calendarId: string
  lastUpdated?: string
}

class ConfigManager {
  private configPath: string
  private userConfig: FeishuUserConfig | null = null

  constructor() {
    const userDataPath = app.getPath('userData')
    this.configPath = path.join(userDataPath, 'feishu-user-config.json')
  }

  /**
   * 加载用户配置
   */
  async loadUserConfig(): Promise<FeishuUserConfig | null> {
    if (this.userConfig) {
      return this.userConfig
    }

    try {
      if (fs.existsSync(this.configPath)) {
        const data = fs.readFileSync(this.configPath, 'utf-8')
        this.userConfig = JSON.parse(data)
        console.log('✅ 加载用户配置成功')
        return this.userConfig
      }
    } catch (error) {
      console.error('加载用户配置失败:', error)
    }
    return null
  }

  /**
   * 保存用户配置
   */
  async saveUserConfig(config: FeishuUserConfig): Promise<boolean> {
    try {
      config.lastUpdated = new Date().toISOString()
      
      // 确保目录存在
      const dir = path.dirname(this.configPath)
      if (!fs.existsSync(dir)) {
        fs.mkdirSync(dir, { recursive: true })
      }
      
      fs.writeFileSync(this.configPath, JSON.stringify(config, null, 2))
      this.userConfig = config
      console.log('✅ 用户配置已保存:', this.configPath)
      return true
    } catch (error) {
      console.error('保存用户配置失败:', error)
      return false
    }
  }

  /**
   * 获取用户配置
   */
  getConfig(): FeishuUserConfig | null {
    return this.userConfig
  }

  /**
   * 检查是否有用户配置
   */
  hasUserConfig(): boolean {
    return this.userConfig !== null && 
           this.userConfig.appId !== '' &&
           this.userConfig.appId !== 'YOUR_APP_ID_HERE'
  }

  /**
   * 删除用户配置（重置）
   */
  async deleteUserConfig(): Promise<void> {
    try {
      if (fs.existsSync(this.configPath)) {
        fs.unlinkSync(this.configPath)
      }
      this.userConfig = null
      console.log('✅ 已删除用户配置')
    } catch (error) {
      console.error('删除用户配置失败:', error)
    }
  }
}

export const configManager = new ConfigManager()
```

### 3. 修改主进程

```typescript
// src/main/index.ts
import { configManager } from './configManager'
import { isDefaultConfig } from './feishuConfig'

let mainWindow: BrowserWindow | null = null

async function createWindow() {
  // 加载用户配置
  const userConfig = await configManager.loadUserConfig()
  
  const win = new BrowserWindow({
    // ... 窗口配置
  })

  // 检查是否需要配置向导
  const needsConfig = !userConfig || isDefaultConfig()
  
  win.loadURL(MAIN_WINDOW_URL)
  
  win.webContents.on('did-finish-load', () => {
    if (needsConfig) {
      // 通知渲染进程显示配置向导
      win.webContents.send('feishu:needsConfig')
    }
  })

  mainWindow = win
}

// IPC 处理器
ipcMain.handle('feishu:saveConfig', async (event, config) => {
  const success = await configManager.saveUserConfig(config)
  if (success && mainWindow) {
    mainWindow.webContents.send('feishu:configSaved', config)
  }
  return { success }
})

ipcMain.handle('feishu:getConfig', async () => {
  return configManager.getConfig()
})

ipcMain.handle('feishu:hasConfig', async () => {
  return configManager.hasUserConfig()
})

ipcMain.handle('feishu:testConfig', async (event, config) => {
  try {
    const result = await testFeishuConfig(config)
    return { success: true, ...result }
  } catch (error) {
    return { success: false, error: error.message }
  }
})

ipcMain.handle('feishu:resetConfig', async () => {
  await configManager.deleteUserConfig()
  return { success: true }
})
```

### 4. 配置向导组件

```tsx
// src/renderer/src/components/FeishuConfigWizard.tsx
import { useState, useEffect } from 'react'

interface FeishuConfigWizardProps {
  onComplete?: () => void
}

export function FeishuConfigWizard({ onComplete }: FeishuConfigWizardProps) {
  const [config, setConfig] = useState({
    appId: '',
    appSecret: '',
    calendarId: ''
  })
  const [saving, setSaving] = useState(false)
  const [testing, setTesting] = useState(false)

  const handleSave = async () => {
    if (!config.appId || !config.appSecret || !config.calendarId) {
      alert('请填写完整的配置信息')
      return
    }

    setSaving(true)
    try {
      await window.api.feishu.saveConfig(config)
      alert('配置保存成功！')
      onComplete?.()
    } catch (error) {
      alert('保存失败：' + error.message)
    } finally {
      setSaving(false)
    }
  }

  const handleTest = async () => {
    setTesting(true)
    try {
      const result = await window.api.feishu.testConfig(config)
      if (result.success) {
        alert('配置有效！可以保存了。')
      } else {
        alert('配置无效：' + result.error)
      }
    } catch (error) {
      alert('测试失败：' + error.message)
    } finally {
      setTesting(false)
    }
  }

  return (
    <div className="fixed inset-0 bg-black bg-opacity-50 flex items-center justify-center z-50">
      <div className="bg-white rounded-lg p-8 max-w-md w-full mx-4">
        <h2 className="text-2xl font-bold mb-4">⚙️ 飞书配置</h2>
        
        <div className="mb-4 p-3 bg-blue-50 rounded text-sm">
          💡 首次使用需要配置飞书凭证
          <br/>
          配置后将自动保存，下次启动无需重新配置
        </div>

        <div className="space-y-4">
          <div>
            <label className="block text-sm font-medium mb-1">
              App ID <span className="text-red-500">*</span>
            </label>
            <input
              type="text"
              value={config.appId}
              onChange={(e) => setConfig({...config, appId: e.target.value})}
              placeholder="cli_xxxxxxxxxxxxxx"
              className="w-full p-2 border rounded focus:ring-2 focus:ring-blue-500"
            />
          </div>

          <div>
            <label className="block text-sm font-medium mb-1">
              App Secret <span className="text-red-500">*</span>
            </label>
            <input
              type="password"
              value={config.appSecret}
              onChange={(e) => setConfig({...config, appSecret: e.target.value})}
              placeholder="请输入 App Secret"
              className="w-full p-2 border rounded focus:ring-2 focus:ring-blue-500"
            />
          </div>

          <div>
            <label className="block text-sm font-medium mb-1">
              Calendar ID <span className="text-red-500">*</span>
            </label>
            <input
              type="text"
              value={config.calendarId}
              onChange={(e) => setConfig({...config, calendarId: e.target.value})}
              placeholder="feishu.cn_xxxxx@group.calendar.feishu.cn"
              className="w-full p-2 border rounded focus:ring-2 focus:ring-blue-500"
            />
          </div>

          <div className="flex gap-2 pt-4">
            <button
              onClick={handleTest}
              disabled={testing || !config.appId || !config.appSecret}
              className="flex-1 px-4 py-2 bg-green-500 text-white rounded hover:bg-green-600 disabled:bg-gray-400"
            >
              {testing ? '测试中...' : '🧪 测试配置'}
```

### 5. 集成到主应用

```tsx
// src/renderer/src/App.tsx
import { useState, useEffect } from 'react'
import { FeishuConfigWizard } from './components/FeishuConfigWizard'

function App() {
  const [needsConfig, setNeedsConfig] = useState(false)

  useEffect(() => {
    // 监听主进程的配置需求通知
    window.api?.feishu?.onNeedsConfig?.(() => {
      setNeedsConfig(true)
    })

    // 检查是否已有配置
    window.api?.feishu?.hasConfig?.().then((has: boolean) => {
      if (!has) {
        setNeedsConfig(true)
      }
    })
  }, [])

  const handleConfigComplete = () => {
    setNeedsConfig(false)
    // 重新加载应用或刷新
    window.location.reload()
  }

  return (
    <div className="App">
      {needsConfig && (
        <FeishuConfigWizard onComplete={handleConfigComplete} />
      )}
      
      {/* 其他应用内容 */}
      <CalendarApp />
    </div>
  )
}
```

## 📝 总结

### 方案优势

1. ✅ **一次配置，永久使用** - 持久化保存到用户数据目录
2. ✅ **不打包配置** - release 中只有示例值
3. ✅ **首次启动引导** - 强制用户配置
4. ✅ **开发友好** - 开发时可用 feishuConfig.ts
5. ✅ **向后兼容** - 保持现有接口不变

### 配置优先级

```
用户配置 (feishu-user-config.json) > 示例配置 (feishuConfig.ts)
```

### 使用场景

* **开发**：修改 feishuConfig.ts 或通过界面配置

* **用户**：首次启动弹出向导，后续自动加载

* **打包**：只打包示例配置，用户配置不打包
