# 修复附件路径去重问题

## 问题诊断

### 现象
导出的附件仍然重复：
```json
"attachments": [
  {
    "path": "C:\\Users\\Administrator\\Documents\\红庄小区 2-1604 房屋租赁合同 (1).docx",
    "name": "Documents 红庄小区 2-1604 房屋租赁合同 (1).docx"
  },
  {
    "path": "\\Users\\Administrator\\Documents\\红庄小区 2-1604 房屋租赁合同 (1).docx",
    "name": "Documents 红庄小区 2-1604 房屋租赁合同 (1).docx"
  }
]
```

### 根本原因

1. **正则匹配问题**：
   - 第一个正则 `/([A-Za-z]:\\[^\s<>"|?*]+)/g` 匹配了 `C:\Users\...`
   - 第二个正则 `/(\.?[\/\\][^\s<>"|?*]+\.[^\s<>"|?*]+)/g` 匹配了 `\Users\...`（作为相对路径）

2. **标准化路径不完整**：
   - `C:\Users\...` → `c:/users/...`
   - `\Users\...` → `/users/...`
   - 两者被认为是不同的路径

3. **文件名提取错误**：
   - 提取的是 `Documents 红庄小区 2-1604 房屋租赁合同 (1).docx` 而不是 `红庄小区 2-1604 房屋租赁合同 (1).docx`
   - 说明 `split(/[\/\\]/)` 没有正确处理中文路径

## 解决方案

### 方案 1: 增强路径标准化（推荐）

**思路**：将不带盘符的路径统一视为与带盘符的路径相同

```typescript
function normalizePath(path: string): string {
  // 1. 移除盘符（C:、D: 等）
  let normalized = path.replace(/^[A-Za-z]:/, '')
  
  // 2. 统一斜杠方向
  normalized = normalized.replace(/\\/g, '/')
  
  // 3. 移除开头的斜杠（/Users -> Users）
  normalized = normalized.replace(/^\/+/, '')
  
  // 4. 转为小写
  normalized = normalized.toLowerCase()
  
  return normalized
}
```

**效果**：
- `C:\Users\Administrator\Documents\file.docx` → `users/administrator/documents/file.docx`
- `\Users\Administrator\Documents\file.docx` → `users/administrator/documents/file.docx`
- ✅ 两者相同，成功去重

### 方案 2: 优化正则表达式顺序

**思路**：确保先匹配完整路径，再匹配不完整路径

```typescript
const patterns = [
  // URL 链接
  /(https?:\/\/[^\s<>"{}|\\^\`\[\]]+)/g,
  // Windows 路径（反斜杠，优先匹配完整路径）
  /([A-Za-z]:\\[^\s<>"|?*]+)/g,
  // Windows 路径（正斜杠）
  /([A-Za-z]:\/[^\s<>"|?*]+)/g,
  // UNC 路径
  /(\\\\[^\s<>"|?*]+)/g,
  // 相对路径（更严格的模式，要求至少两级目录）
  /(\.[\/\\](?:[^\s<>"|?*]+[\/\\])+[^\s<>"|?*]+\.[^\s<>"|?*]+)/g,
  // 绝对路径的剩余部分（最后匹配，排除已匹配的内容）
  /(?<![A-Za-z:])(\\[^\s<>"|?*]+(?:\\[^\s<>"|?*]+)+\.[^\s<>"|?*]+)/g
]
```

**缺点**：正则表达式复杂，难以维护

### 方案 3: 后处理去重

**思路**：在提取完所有附件后，根据文件名和路径相似度去重

```typescript
// 在 extractAttachments 函数末尾添加
function deduplicateAttachments(attachments: Attachment[]): Attachment[] {
  const seen = new Map<string, Attachment>()
  
  for (const attachment of attachments) {
    // 生成去重键：文件名 + 路径哈希
    const key = `${attachment.name}_${normalizePath(attachment.path)}`
    
    if (!seen.has(key)) {
      seen.set(key, attachment)
    }
  }
  
  return Array.from(seen.values())
}
```

### 第二步：修复文件名提取

当前文件名提取错误：
- 输入：`C:\Users\Administrator\Documents\红庄小区 2-1604 房屋租赁合同 (1).docx`
- 输出：`Documents 红庄小区 2-1604 房屋租赁合同 (1).docx` ❌
- 期望：`红庄小区 2-1604 房屋租赁合同 (1).docx` ✅

**问题原因**：路径中的反斜杠可能被转义了

**修复方案**：
```typescript
function extractFileName(path: string): string {
  // 1. 标准化路径（统一斜杠）
  const normalized = path.replace(/\\/g, '/')
  
  // 2. 分割路径
  const parts = normalized.split('/')
  
  // 3. 取最后一部分
  return parts[parts.length - 1] || normalized
}
```

function isValidPath(path: string): boolean {
  // 检查是否包含至少一个反斜杠或正斜杠

}
