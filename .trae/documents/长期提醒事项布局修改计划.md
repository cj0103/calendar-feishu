# 长期提醒事项布局修改计划

## 尺寸测算

**窗口尺寸**：1200px 宽，800px 高

**高度分配**：
- 顶部导航栏：约 32px
- 月份切换栏：约 30px
- 星期标题：约 25px
- 日历4周：min-h-[500px]，每周约 125px
- 长期提醒区域可用高度：约 800 - 32 - 30 - 25 - 500 - 16 = **约 197px**

**左侧事项区域**（4/7 宽度 = 约686px）：
- 内部再分2列，每列约 343px
- 每个事项行高约 28-30px
- **每列可放约 6 个事项**

**结论**：
- 左列：事项1-6
- 右列：事项7-12
- 超过12个事项后，提示用户删除部分事项

```
┌──────────────────────────────────────┬─────────────────────────┐

│ 事项1（描述内容）  │    事项7（描述内容）│                         │

│ 事项4（描述内容）  │    事项10（描述内容）│                         │
│ 事项5（描述内容）  │    事项11（描述内容）│                         │
│ 事项6（描述内容）  │    事项12（描述内容）│                         │
└───────────────────┴──────────────────┴─────────────────────────┘
```

## 布局结构

### 外层：左右结构
- **左侧**：长期提醒事项区域，占 4/7 宽度
- **右侧**：待升级区域，占 3/7 宽度

### 左侧内部：2列布局
- **左列**：事项1-6
- **右列**：事项7-12
- 每列事项连续排列，一行一行显示

### 事项显示方式
- 每个事项占一行，不分成格子
- 行高约 28-30px
- 背景色根据重要性区分（高=淡红、中=淡橙、低=默认）
- 标题和描述合并：`标题（描述）`
- 双击空白区域新增，双击事项编辑，右击事项弹出菜单
- 超过12个事项时提示用户

## 实现步骤

### 步骤1：修改 LongTermRemindersPanel.tsx

1. 外层使用 `flex` 布局，左侧事项区域，右侧待升级区域
2. 左侧区域内部使用2列布局
3. 将事项数组分成两半：前6个放左列，后6个放右列
4. 每列事项垂直排列，每个事项一行
5. 设置固定高度（约197px），避免滚动条
6. 右侧待升级区域占满剩余空间
7. 超过12个事项时显示提示
