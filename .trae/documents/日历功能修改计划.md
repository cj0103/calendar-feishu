# 日历功能修改计划

## 问题1：周末背景颜色显示不正确

### 问题分析
在 `App.tsx` 的背景颜色逻辑中（约930-947行），颜色判断顺序是：
1. 调休工作日 → workdayColor
2. 法定节假日 → weekendColor  
3. **非当月 → otherMonthColor** ← 问题在这里
4. 周末 → weekendColor
5. 工作日 → workdayColor

改为4周（28天）后，很多周末格子同时满足"非当月"和"周末"两个条件，由于"非当月"判断在前，导致周末背景被非当月背景覆盖了。

### 修改方案
调整判断优先级，**周末判断应该在非当月之前**：
1. 调休工作日 → workdayColor
2. 法定节假日 → weekendColor
3. **周末 → weekendColor** ← 提前到这里
4. 非当月 → otherMonthColor
5. 工作日 → workdayColor

这样即使是跨月的周末，也会优先应用周末背景色。

## 问题2：长期提醒事项区域优化

### 尺寸测算

**窗口尺寸**：1200px 宽，800px 高
**日历区域**：7列栅格，每列约 160px
**日历高度**：min-h-[500px]，4周均分，每周格子约 120px

**长期事项区域布局**：
- 采用7列栅格，但**只使用左侧4列**显示事项
- 右侧3列留空，标注"待升级"
- 每个事项格子：宽度约 160px，高度约 120px（与日历每周格子一致）
- 4列 × 3行 = 12个事项格子，空间充足

### 2.1 布局结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 长期提醒事项  (双击新增，右击编辑/删除)                                 │
├──────────┬──────────┬──────────┬──────────┬─────────────────────────────┤
│ 事项1    │ 事项2    │ 事项3    │ 事项4    │                             │
│ (描述)   │ (描述)   │ (描述)   │ (描述)   │                             │
├──────────┼──────────┼──────────┼──────────┤        待升级               │
│ 事项5    │ 事项6    │ 事项7    │ 事项8    │                             │
│          │          │          │          │      (预留区域)             │
├──────────┼──────────┼──────────┼──────────┤                             │
│ 事项9    │ 事项10   │ 事项11   │ 事项12   │                             │
│          │          │          │          │                             │
└──────────┴──────────┴──────────┴──────────┴─────────────────────────────┘
```

### 2.2 格子高度

设置为 `min-h-[120px]`，与日历每周格子高度一致。

### 2.3 格子内容

- 标题和描述合并为一行：`标题（描述）`
- 超长文本自动截断（truncate）
- 背景色根据重要性显示（高=红、中=橙、低=灰/默认）
- **不显示重要性文字标注**

### 2.4 交互方式

- **双击空白格子** → 新增事项
- **双击已有事项** → 编辑事项
- **右击事项** → 弹出菜单（编辑/删除）
- 删除所有操作按钮（✏️🗑️和"新增"按钮）

### 2.5 待升级区域

- 右侧3列留空
- 显示"待升级"提示文字
- 半透明灰色背景，与事项格子区分
