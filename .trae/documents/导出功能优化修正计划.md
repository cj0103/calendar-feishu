# 导出功能优化修正计划

## 问题分析

### 问题 1: 冗余字段

用户指出以下字段可以通过系统时间倒推，不需要导出：

```json
"dateTime": {
  "dayOfWeek": "星期六",  // ❌ 冗余：可从 date 计算
  "lunarDate": "十九"     // ❌ 冗余：可从 date 计算
}
```

**需要检查的冗余字段**:

1. **dayOfWeek**: 从 `date` 可以计算
2. **lunarDate**: 从 `date` 可以计算
3. **endTime**: 格式不一致（有时是时间，有时是 ISO 日期）
4. **syncInfo.lastSyncTime**: 时间戳格式，大模型不易读
5. **statistics.byImportance**: 可遍历 events 统计
6. **statistics.byMonth**: 可遍历 events 统计
7. **statistics.attachmentTypes**: 可遍历 attachments 统计

### 问题 2: 附件重复导出

**现象**: 同一个路径被识别为两个附件
```json
"attachments": [
  {
    "path": "C:\\Users\\Administrator\\Documents\\红庄小区 2-1604 房屋租赁合同 (1).docx",
    "name": "Documents 红庄小区 2-1604 房屋租赁合同 (1).docx",  // ❌ 文件名提取错误
    "type": "document"
  },
  {
    "path": "\\Users\\Administrator\\Documents\\红庄小区 2-1604 房屋租赁合同 (1).docx",  // ❌ 路径不完整
    "name": "Documents 红庄小区 2-1604 房屋租赁合同 (1).docx",  // ❌ 文件名提取错误
    "type": "document"
  }
]
```

**原因分析**:

1. **正则匹配问题**: 可能同时匹配了完整路径和部分路径
2. **文件名提取错误**: `split(/[\/\\]/).pop()` 提取的不是最后的文件名
3. **路径去重缺失**: 没有检查重复的路径

**根本原因**: 正则表达式匹配了 `C:\path` 和 `\path` 两次

## 优化方案

### 方案 1: 精简冗余字段（推荐）

**保留核心字段**:
```json
{
  "exportInfo": {
    "exportedAt": "2026-03-14T10:30:00.000Z",
    "dateRange": { "start": "2026-01-01", "end": "2026-12-31" },
    "totalEvents": 150,
    "withAttachments": 45,
    "formatVersion": "2.0"
  },
  "events": [
    {
      "id": "1234567890",
      "dateTime": {
        "date": "2026-03-15",
        "startTime": "14:30",
        "endTime": "15:30"  // 保持原始格式
      },
      "basicInfo": {
        "title": "会议",
        "location": "会议室",
        "importance": "high"
      },
      "content": {
        "description": "文字描述",
        "attachments": [
          {
            "path": "C:\\Documents\\file.docx",
            "name": "file.docx",
            "type": "document"
          }
        ],
        "rawDescription": "原始内容"
      },
      "syncInfo": {
        "source": "local",
        "feishuEventId": null
      }
    }
  ]
}
```

**删除字段**:
- ❌ `dateTime.dayOfWeek` - 可计算
- ❌ `dateTime.lunarDate` - 可计算
- ❌ `syncInfo.lastSyncTime` - 内部使用
- ❌ `statistics` - 可遍历计算

**保留理由**:
- ✅ `endTime` - 结束时间，用户输入，无法倒推
- ✅ `syncInfo.source` - 判断数据来源
- ✅ `syncInfo.feishuEventId` - 飞书 ID，用于后续同步

### 方案 2: 修复附件重复问题

**修改 extractAttachments 函数**:

```typescript
export function extractAttachments(description: string): ParseResult {
  const attachments: Attachment[] = []
  const seenPaths = new Set<string>()  // 去重
  
  let text = description
  
  // 正则表达式（优化顺序）
  const patterns = [
    // URL 链接
    /(https?:\/\/[^\s<>"{}|\\^\`\[\]]+)/g,
    // Windows 路径（反斜杠，优先匹配完整路径）
    /([A-Za-z]:\\[^\s<>"|?*]+)/g,
    // Windows 路径（正斜杠）
    /([A-Za-z]:\/[^\s<>"|?*]+)/g,
    // UNC 路径
    /(\\\\[^\s<>"|?*]+)/g,
    // 相对路径（最后匹配，避免与绝对路径冲突）
    /(\.?[\/\\](?:[^\s<>"|?*]+[\/\\])+[^\s<>"|?*]+\.[^\s<>"|?*]+)/g
  ]
  
  for (const pattern of patterns) {
    const matches = description.match(pattern) || []
    for (const match of matches) {
      // 标准化路径（去重）
      const normalizedPath = normalizePath(match)
      
      // 跳过已处理的路径
      if (seenPaths.has(normalizedPath)) {
        continue
      }
      seenPaths.add(normalizedPath)
      
      // 提取文件名（正确方式）
      const name = extractFileName(match)
      
      // 判断类型
      const type = getAttachmentType(match)
      
      attachments.push({
        path: match,  // 保留原始路径
        name,
        type
      })
      
      // 从文本中移除路径
      text = text.replace(match, '')
    }
  }
  
  // 清理多余空白
  text = text.replace(/  +/g, ' ').trim()
  
  return { text, attachments }
}

/**
 * 标准化路径（用于去重）
 */
function normalizePath(path: string): string {
  // 统一斜杠方向
  return path.replace(/\\/g, '/').toLowerCase()
}

/**
 * 正确提取文件名
 */
function extractFileName(path: string): string {
  // 移除协议前缀
  let cleanPath = path.replace(/^https?:\/\//, '')
  
  // 提取最后一部分
  const parts = cleanPath.split(/[\/\\]/)
  return parts[parts.length - 1] || cleanPath
}
```

        "description": "文字描述",
        "attachments": [
          {
            "path": "C:\\Documents\\file.docx",

        ],
        "rawDescription": "原始内容（兼容用，可选）"
      },

### 修改清单

**删除字段**:
1. ❌ `dateTime.dayOfWeek`
2. ❌ `dateTime.lunarDate`
3. ❌ `syncInfo.lastSyncTime`
4. ❌ `statistics` (整个对象)

**修改逻辑**:
1. ✅ 修复附件去重逻辑
2. ✅ 修复文件名提取逻辑
3. ✅ 优化正则表达式顺序

**保留字段说明**:
- ✅ `endTime`: 用户输入的结束时间，无法从其他字段倒推
- ✅ `rawDescription`: 保留原始内容，便于兼容和调试
- ✅ `syncInfo.source`: 判断数据来源（本地/飞书）
- ✅ `syncInfo.feishuEventId`: 用于后续同步
