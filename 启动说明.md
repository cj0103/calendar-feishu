# 🚀 隐藏终端启动开发服务器指南

## 📋 问题描述
开发阶段使用 `npm run dev` 启动应用时，终端窗口显示在任务栏，容易被误关闭。

## 💡 快速开始

**最简单的方法**：双击 `start-dev.vbs` 文件即可！

## 🔧 解决方案

### ✅ 方案 1：VBScript 脚本（最简单，推荐）

**使用方法**：
1. 双击项目根目录的 `start-dev.vbs` 文件
2. 开发服务器会在后台启动
3. 终端窗口不会显示在任务栏

**优点**：
- 简单方便，双击即可
- 完全隐藏终端窗口
- 不影响应用正常运行

**缺点**：
- 无法直接看到控制台输出
- 需要查看日志时需使用其他方式

**停止服务**：
- 打开任务管理器（Ctrl+Shift+Esc）
- 找到 `node.exe` 进程
- 结束进程即可

---

### ✅ 方案 2：PowerShell 脚本

**使用方法**：
1. 右键点击 `start-dev.ps1` 文件
2. 选择"使用 PowerShell 运行"

**优点**：
- PowerShell 原生支持
- 脚本更简洁

**缺点**：
- 需要右键选择运行
- 默认 PowerShell 策略可能阻止脚本执行

**如果遇到权限问题**：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

---

### ✅ 方案 3：VS Code 任务配置（开发推荐）

**使用方法**：
1. 在 VS Code 中按 `Ctrl+Shift+P`
2. 输入 "Tasks: Run Task"
3. 选择 "npm: dev"

**配置说明**：
已在 `.vscode/tasks.json` 中配置好后台任务

**优点**：
- 集成在 VS Code 中
- 可以查看控制台输出
- 不会被误关闭
- 支持快捷键启动

**快捷键启动**：
可以配置自定义快捷键来运行任务

**停止服务**：
- 在 VS Code 终端中按 `Ctrl+C`
- 或关闭 VS Code

---

### ✅ 方案 4：创建桌面快捷方式

**步骤**：
1. 右键点击 `start-dev.vbs`
2. 选择"发送到" → "桌面快捷方式"
3. 可以重命名为"启动开发服务器"
4. 双击桌面快捷方式即可启动

**优点**：
- 快速启动
- 可以设置开机自启
- 可以自定义图标

---

## 📝 注意事项

1. **查看日志**：
   - 隐藏终端后无法直接看到日志
   - 建议同时打开 VS Code 的输出面板
   - 或使用 `npm run dev > output.log 2>&1` 重定向输出

2. **停止服务**：
   - 使用任务管理器结束 `node.exe` 进程
   - 或在 VS Code 中停止（如果使用方案 3）

3. **端口占用**：
   - 如果启动失败，检查端口是否被占用
   - 默认端口通常是 3000 或 5173（Vite）

4. **环境变量**：
   - 确保 `.env` 文件配置正确
   - 隐藏终端不影响环境变量加载

## 🔧 高级配置

### 重定向输出到日志文件

修改 `start-dev.vbs`：
```vbscript
Set objShell = CreateObject("WScript.Shell")
' 重定向输出到日志文件
objShell.Run "cmd /c npm run dev > dev.log 2>&1", 0, False
```

### 使用 NSSM 创建 Windows 服务（高级）

适合需要长期运行的场景：
```bash
# 安装 nssm
nssm install CalendarDev "C:\Program Files\nodejs\node.exe"
nssm set CalendarDev ApplicationParameters "node_modules/.bin/vite"
nssm set CalendarDev AppDirectory "d:\Ollama\trae_pro\calendar-feishu"
nssm start CalendarDev
```

## ❓ 常见问题

**Q: 启动后看不到任何反应？**
A: 这是正常的，服务已经在后台启动。可以打开浏览器访问 `http://localhost:5173` 确认。

**Q: 如何确认服务是否启动成功？**
A: 
1. 打开浏览器访问 `http://localhost:5173`
2. 或使用 `netstat -ano | findstr :5173` 查看端口占用

**Q: 脚本无法运行？**
A: 
1. 确保已安装 Node.js 和 npm
2. 确保在项目根目录运行
3. 检查 PowerShell 执行策略

**Q: 如何修改启动端口？**
A: 修改 `vite.config.ts` 或 `.env` 文件中的端口配置。
